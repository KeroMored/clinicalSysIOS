# نظام الموافقة على الجيمات

## نظرة عامة

تم إضافة نظام موافقة كامل للجيمات مع فلترة حسب الحالة وإمكانية التحكم الكامل من لوحة الأدمن.

## الميزات الرئيسية

### 1. الحالات المختلفة

- **قيد الانتظار (Pending)**: الجيمات الجديدة التي لم يتم الموافقة عليها
- **موافق عليها (Approved)**: الجيمات المعتمدة والمنشورة للمستخدمين
- **الكل (All)**: عرض جميع الجيمات بغض النظر عن حالتها

### 2. إضافة جيم جديد

- عند إضافة جيم جديد من `AddGymScreen`:
  - يتم تعيين `isApproved = false` تلقائياً
  - لا يتم تعيين `approvedAt` حتى الموافقة عليه
  - يظهر رسالة للمستخدم تفيد بأن الجيم سيتم مراجعته قبل النشر

### 3. شاشة الموافقة على الجيمات (`GymApprovalScreen`)

تتيح للأدمن:

#### الفلترة

- **قيد الانتظار**: عرض الجيمات التي تنتظر الموافقة
- **موافق عليها**: عرض الجيمات المعتمدة
- **الكل**: عرض جميع الجيمات

#### الإجراءات المتاحة

- **الموافقة**: تحويل الجيم من قيد الانتظار إلى موافق عليه
  - يتم تعيين `isApproved = true`
  - يتم تعيين `approvedAt = timestamp الحالي`
  - يصبح الجيم مرئياً للمستخدمين

- **إرجاع للانتظار**: تحويل الجيم من موافق عليه إلى قيد الانتظار
  - يتم تعيين `isApproved = false`
  - يتم حذف `approvedAt`
  - يختفي الجيم من قائمة المستخدمين

- **رفض وحذف**: حذف الجيم نهائياً من النظام
  - يتم حذف السجل بالكامل من Firestore
  - لا يمكن التراجع عن هذا الإجراء

- **عرض التفاصيل**: فتح شاشة تفاصيل الجيم الكاملة

### 4. عرض الجيمات للمستخدمين

- شاشة `GymsListScreen` تعرض فقط الجيمات الموافق عليها
- تستخدم `loadApprovedGyms()` من الـ Cubit
- Repository يفلتر تلقائياً بـ `isApproved = true` و `isActive = true`

## الملفات المعدلة

### 1. النموذج (Model)

**File**: `gym_model.dart`

- يحتوي بالفعل على الحقول:
  - `isApproved`: حالة الموافقة
  - `approvedAt`: تاريخ الموافقة

### 2. المستودع (Repository)

**File**: `gym_repository.dart`

- `getApprovedGyms()`: يعيد فقط الجيمات الموافق عليها
- `getPendingGyms()`: يعيد الجيمات قيد الانتظار
- `approveGym(id)`: تحديث حالة الجيم للموافقة عليه
- `deleteGym(id)`: حذف الجيم

### 3. الشاشات الجديدة

**File**: `gym_approval_screen.dart`

- شاشة إدارة كاملة للموافقات
- فلترة بـ Filter Chips
- كروت تعرض معلومات الجيم
- أزرار إجراء مخصصة حسب الحالة

### 4. التعديلات على الملفات الموجودة

**File**: `add_gym_screen.dart`

- تغيير `isApproved: false` (كان `true`)
- حذف `approvedAt` من الإنشاء الأولي
- تحديث رسالة النجاح لتوضيح أن الجيم يحتاج موافقة

**File**: `admin_home_page.dart`

- تحديث الـ import لاستخدام `GymApprovalScreen` الجديدة
- إزالة الحاجة لـ BlocProvider عند فتح الشاشة

## كيفية الاستخدام

### للأدمن

1. من إدارة الأدمن، اضغط على "الموافقة على الجيمات"
2. استخدم الفلاتر لعرض الجيمات حسب الحالة
3. اضغط على "عرض التفاصيل" لرؤية المعلومات الكاملة
4. اضغط على:
   - "موافقة" للموافقة على الجيم
   - "إرجاع للانتظار" لإلغاء الموافقة
   - أيقونة "X" لرفض وحذف الجيم

### للمستخدم

1. أضف جيم جديد من الشاشة المخصصة
2. ستظهر رسالة تفيد بأن الجيم سيتم مراجعته
3. لن يظهر الجيم في القائمة حتى موافقة الأدمن
4. بعد الموافقة، يصبح الجيم مرئياً في قائمة الجيمات

## قاعدة البيانات (Firestore)

### حقول الجيم

```dart
{
  "name": "اسم الجيم",
  "description": "الوصف",
  // ... باقي الحقول
  "isApproved": false,        // false = قيد الانتظار, true = موافق عليه
  "isActive": true,           // للتحكم في التفعيل/التعطيل
  "createdAt": Timestamp,     // تاريخ الإضافة
  "approvedAt": Timestamp?,   // تاريخ الموافقة (null إذا لم يتم الموافقة)
}
```

### Queries المستخدمة

```dart
// للمستخدمين - فقط الموافق عليها
.where('isApproved', isEqualTo: true)
.where('isActive', isEqualTo: true)

// للأدمن - قيد الانتظار
.where('isApproved', isEqualTo: false)

// للأدمن - موافق عليها
.where('isApproved', isEqualTo: true)

// للأدمن - الكل (بدون filter)
```
